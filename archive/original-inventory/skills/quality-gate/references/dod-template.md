# Definition of Done (DoD) Template

Use these as `checks[]`.

## Core
- [ ] `C-CORE-1` Problem solved for stated scope.
- [ ] `C-CORE-2` No known blocking correctness/safety defects.
- [ ] `C-CORE-3` Failure paths and edge cases handled.

## Simplicity and Consistency
- [ ] `C-SIMPLE-1` Design kept minimal for current requirements.
- [ ] `C-SIMPLE-2` Interfaces and naming stay consistent.
- [ ] `C-SIMPLE-3` Unnecessary complexity removed.

## Verification
- [ ] `C-VERIFY-1` Automated tests cover key paths and regressions.
- [ ] `C-VERIFY-2` Required checks completed and passing.
- [ ] `C-VERIFY-3` Residual risks documented.
- [ ] `C-VERIFY-4` CodeRabbit prompt-only committed review triaged; actionable findings resolved or explicitly waived.
- [ ] `C-VERIFY-5` Standard application verification commands completed for the target repo.

## Documentation
- [ ] `C-DOC-1` Docs/specs updated for behavior changes.
- [ ] `C-DOC-2` Follow-ups captured explicitly.
